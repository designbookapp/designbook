Designbook **injects into your app's own Vite dev server** and opens your running app inside
one full view — every discipline's tools around the same live app, all derived from the same
code. Designers edit theme tokens, writers edit every string in every locale, product flips
per-tenant flags, engineers see the source. A pencil button opens the full view right on your
running app; a play button in the same spot returns you to it, untouched. An embedded coding
agent applies bigger changes as real edits to your source files, staged so you can review them
before anything lands on your branch.

<CardGrid>
  <Card title="Real components, real code" icon="seti:react">
    Your components hot-reload exactly as they do in your app — because it is your app. Edits
    the agent makes land as commit-ready file changes, not throwaway mockups.
  </Card>
  <Card title="Chat that remembers" icon="comment">
    Select anything in your running app and ask — the agent gets your selection as context and
    keeps memory across everything you select and discuss in one conversation.
  </Card>
  <Card title="Changesets, not surprises" icon="approve-check">
    Every exploration is isolated as a hidden git branch until you bake it in, branch it out for
    review, or discard it. Your real branch stays pristine the whole time.
  </Card>
  <Card title="Designer-facing editing" icon="setting">
    Adapters expose theme tokens, translated text, and per-tenant flags as editable fields,
    writing back to your CSS, locale JSON, and flag files.
  </Card>
  <Card title="Your real build pipeline" icon="rocket">
    In injected mode your components compile through your app's own Vite — its aliases,
    tsconfig paths, and providers apply, so previews match production. (Host mode uses an
    embedded Vite with the same bridges auto-detected from your repo.)
  </Card>
  <Card title="Figma round-trip" icon="figma">
    Push components to Figma as native layers with bound variables, then pull designer edits
    back as a target the agent applies to your source.
  </Card>
</CardGrid>
