Designbook **injects into your app's own Vite dev server** and opens the running app as a
set of live, editable views — one per discipline, all derived from the same code. Designers
edit theme tokens, writers edit every string in every locale, product flips per-tenant
flags, engineers see the source. A `◈ designbook` toolbar pill opens tools right on your
running app, or expands into a full-screen canvas where your components render through your
app's own bundler, styling, and providers — and an embedded coding agent applies bigger
changes as real edits to your source files.

<CardGrid>
  <Card title="Real components, real code" icon="seti:react">
    Your components hot-reload exactly as they do in your app. Edits the agent makes land
    as commits-ready file changes, not throwaway mockups.
  </Card>
  <Card title="Edit right on the page" icon="pencil">
    Select a component, prompt Pi, or edit text in place directly on your running app — the
    full canvas is one step further, not the only way in.
  </Card>
  <Card title="Designer-facing editing" icon="setting">
    Adapters expose theme tokens, translated text, and per-tenant flags as editable
    fields on the canvas, writing back to your CSS, locale JSON, and flag files.
  </Card>
  <Card title="Your real build pipeline" icon="approve-check">
    In injected mode your components compile through your app's own Vite — its aliases,
    tsconfig paths, and providers apply, so previews match production. (Host mode uses an
    embedded Vite with the same bridges auto-detected from your repo.)
  </Card>
  <Card title="Figma round-trip" icon="figma">
    Push components to Figma as native layers with bound variables, then pull designer
    edits back as a target the agent applies to your source.
  </Card>
