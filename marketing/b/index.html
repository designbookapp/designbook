Engineers set the boundaries

Empowerment with guardrails. Yours.

Refract doesn’t route around engineering — it routes through it. One config declares what’s editable; everything else stays untouchable.

refract.config.ts
// what the team can touch — nothing else
export default {
  surfaces: {
    theme:  "theme/tokens.json",
    copy:   "copy/*.json",     // all locales
    flags:  "flags/tenants.yaml"
  },
  review: {
    required: true,            // always
    codeowners: "@acme/frontend"
  }
}
Code stays the source of truthNo shadow copy, no export step. Surfaces are views into the repo itself.
Every change is a diffVisual edit or agent edit — it lands as a pull request with a full trail. Nothing merges without your review.
Boundaries in one fileDeclare the editable surfaces once. Widen them when trust grows, tighten them when it matters.

Getting started: connect your GitHub repo and the agent reads the codebase and drafts this config for you. You approve every line before anyone edits anything.
