SPEC-001 · General notes

Derived from code. Never drawn.

The set is not a parallel artifact that drifts. It's a projection of the running app — current by construction, bounded by config your engineers own.

runs inside your real app

An npm plugin in your build: real components, real data, live routes. No storybook-of-lies to keep honest.

surfaces: one config

Adapters declare what each sheet exposes — theme tokens, locale catalogs, flag providers, datasets, variants, flows. Everything else is read-only by construction.

every edit lands as code

The embedded agent reconciles edits into the repo: a CSS custom property, a locale-file diff, a feature-flag JSON change. Reviewable, revertible, boring.

every branch, a set

Each branch renders its own drawing set at a URL. Share a reading, not a screenshot.

figma is a projection

Push any sheet to Figma as native, editable layers; pull the edits back and the agent applies them. The code stays the source.

designbook.config.ts
import { defineBook } from "designbook";

export default defineBook({
  app: "src/App.tsx",          // your real entry
  sheets: {
    theme:   "src/theme/tokens.css",
    content: "locales/*.json",
    flags:   "flags/meridian.json",
    data:    "fixtures/**",
  },
  clash: { locales: "strict", contracts: "flag" },
  reviewers: ["@acme/platform"],
});  // everything above is editable. nothing else is.
