For the engineers

Code stays the source of truth.

boundaries: one config file

You declare exactly which surfaces designers can touch — tokens, strings, flags. Everything else is read-only, by construction.

writes: pull requests only

No hidden database of overrides. Every edit — human or agent — becomes a diff on a branch, gated by your normal review.

audit: full trail

Who changed what, in which surface, applied by which agent run. Exportable, greppable, boring on purpose.

runs on your stack

Your components, your build, your CI. Self-hosting available on Enterprise; the local CLI is open source.

designbook.config.ts 
export default defineBook({
  components: "src/components/**",
  surfaces: {
    theme:  { file: "src/theme.ts" },
    text:   { catalog: "locales/*.json" },
    flags:  { provider: launchDarkly() },
  },
  writes: "pull-request",   // never direct
  reviewers: ["@acme/design-eng"],
})
